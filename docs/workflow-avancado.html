Auto-Discovery de Campos

Antes de escrever o manifesto manualmente, use o /poui-specialist:discover para inspecionar o endpoint e gerar o manifesto automaticamente.

O plugin:

  1. Lê o proxy.conf.json do projeto Angular para obter a URL do AppServer
  2. Chama GET <appserver>/rest/api/custom/v1/titulos?pageSize=1
  3. Inspeciona os campos do primeiro registro — tipos, nomes, padrão de chave composta
  4. Gera o manifesto pré-preenchido com a tripla page-list + page-edit + service
  5. Pergunta se deseja gerar os componentes — se sim, despacha o generate-batch direto

O que é inferido automaticamente

CampoInferência
Nome começa com Cd, Cod, NumMarcado como (req) no page-edit
Nome contém Dt, Dat, DataREGRA: usar po-datepicker
Nome contém Vl, Vlr, Val, ValorREGRA: formatar como moeda BRL
Response tem codigo + lojaChave composta: endpoint /{codigo}/{loja}

Exemplo de saída

Ajuste MODULO e PASTA_DESTINO antes de confirmar (ex: financeiro/titulos). Os demais campos são preenchidos automaticamente.

Screenshot para Código

Se tiver um screenshot, wireframe ou print de sistema legado, use /poui-specialist:screenshot para gerar o manifesto automaticamente por visão — sem endpoint de API e sem escrever o manifesto à mão.

Exemplos:

O plugin:

  1. Lê a imagem via path local ou URL
  2. Analisa com visão multimodal — infere tipo de componente, campos, ações e módulo
  3. Exibe laudo com o que foi detectado + manifesto pré-preenchido
  4. Pergunta se deseja gerar os componentes agora — se sim, despacha o generate-batch direto

Geração Codebase-Aware

Antes de gerar componentes, use /poui-specialist:context para escanear o projeto e evitar duplicatas.

O plugin:

  1. Localiza angular.json subindo até 3 níveis
  2. Lê app.routes.ts e extrai todas as rotas registradas
  3. Lista os serviços gerados pelo plugin (@generated poui-specialist) com seus endpoints
  4. Exibe o snapshot e pergunta se deseja ativá-lo para as próximas gerações

O que o contexto faz na geração

Situação detectadaAção
Rota já registradaAvisa e não duplica no app.routes.ts
Serviço com mesmo endpoint já existeReutiliza o serviço em vez de criar novo arquivo
API_BASE não informado no manifestoSugere o valor inferido dos serviços existentes

Verificação de Build Automática

Após gerar os componentes, o plugin compila o projeto Angular automaticamente e corrige erros TypeScript/template sem intervenção do usuário.

Esta skill é invocada automaticamente pelo /generate e /generate-batch — não é necessário chamá-la manualmente.

Como funciona

  1. Localiza o angular.json subindo até 3 níveis a partir do diretório atual
  2. Executa ng build --configuration development com timeout de 120 segundos
  3. Se o build passou: exibe confirmação e encerra
  4. Se houver erros: lê cada arquivo com erro e aplica correção automática
  5. Repete até o build passar ou esgotar 3 tentativas

Filtro de segurança

O plugin só modifica arquivos que ele mesmo gerou — identificados pelo header @generated poui-specialist. Arquivos pré-existentes do projeto aparecem no relatório final mas não são tocados.

Tipos de erro corrigidos automaticamente

Type 'string' is not assignable to type 'number'Ajusta casting ou tipo na interface
Expected N arguments, but got MAjusta chamada do método

Auditoria de Qualidade

Use /poui-specialist:quality para verificar se os componentes gerados estão seguindo os critérios de qualidade do plugin.

O plugin varre todos os *.component.ts com @generated poui-specialist e aplica 4 regras por componente via regex, mais uma verificação de rotas:

#CritérioO que verificaSeveridade
1OnPushChangeDetectionStrategy.OnPush presente no decoratorCrítico
2Loading statefinalize() presente (evita spinner travado)Atenção
3Error handlingnotification.error() ou catchError() presenteAtenção
4Cleanup observablestakeUntilDestroyed() presente (evita memory leak)Atenção
5Lazy loadingRotas em app.routes.ts usam loadComponent: (auditado separadamente)Atenção
