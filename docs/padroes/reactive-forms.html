Reactive Forms com PO-UI

Padrões para usar Angular Reactive Forms com campos PO-UI em telas po-page-edit.

Quando usar Reactive Forms vs PoDynamicForm:

  • Reactive Forms → validação cruzada entre campos, lógica de disable dinâmica, FormArray para linhas repetíveis, controle preciso sobre o modelo de dados.
  • PoDynamicForm → campos simples sem dependências, prototipagem rápida, menos boilerplate.

patchValue vs setValue vs getRawValue

MétodoComportamentoQuando usar
form.patchValue(obj)Atualiza apenas as propriedades presentes em obj; ignora propriedades ausentesLoad de dados da API (response pode não ter todos os campos)
form.setValue(obj)Exige que obj contenha todos os campos — erro se faltar algumRaramente necessário; prefira patchValue
form.getRawValue()Retorna todos os campos incluindo os disabledSempre usar no submit — não omite campos desabilitados
form.valueRetorna apenas campos enabled; campos disabled são omitidosNunca usar no submit; usar apenas para inspeção
form.markAllAsTouched()Ativa exibição de erros em todos os camposChamar ao tentar salvar com formulário inválido
ctrl.reset(value?)Limpa o campo e marca como pristine + untouchedLimpar campo ao desabilitá-lo por lógica condicional
