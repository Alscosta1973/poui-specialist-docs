/test

Gera unit tests Karma + Jasmine para componentes e services PO-UI Angular

/poui-specialist:test

Gera *.component.spec.ts ou *.service.spec.ts completo (Karma + Jasmine) para componentes e services criados pelo plugin, cobrindo: smoke test, estados de loading/empty/error, chamadas HTTP via HttpTestingController, navegação via Router, e interações específicas por tipo de componente.

Sintaxe

Exemplos

Famílias de componente e cobertura

O gerador detecta automaticamente o tipo do componente e aplica o template de cobertura correspondente:

FamíliaDetectado porSpecs geradosValidado em
listitems + loading + hasNext, PoDisclaimerGroup, PoPageDynamicTableComponentSmoke, carregamento, estado vazio, paginação, filtrosParceirosComponent, TitulosListComponent

other@Injectable, PoChartModule, PoWidgetModuleSmoke, chamadas HTTP, mapeamento de resposta, errosTitulosService

Fluxo de geração

  1. Parse de argumentos — deriva kebab-name, detecta se é service, calcula caminhos de componente e spec
  2. Leitura e análise — lê o arquivo, identifica família, extrai ServiceClass, apiPath e ModelInterface
  3. Seleção de template — carrega templates-test-base.md + templates-test-<família>.md
  4. Geração do spec — substitui placeholders, preenche dados mock realistas a partir das PoTableColumn[]
  5. Execução e relatório — roda ng test --watch=false e exibe resultado
