PO-UI Modal, Dialog e Dynamic View

po-modal

Componente de diálogo modal. Controlado via @ViewChild + métodos open() / close().

Key Inputs

InputTypeDescription
p-titlestringTítulo exibido no cabeçalho
p-size'sm' | 'md' | 'lg' | 'xl' | 'auto'Largura do modal (default 'md')
p-primary-actionPoModalActionBotão primário (confirmar/salvar)
p-secondary-actionPoModalActionBotão secundário (cancelar/fechar)
p-hide-closebooleanOculta o X de fechar

PoModalAction

Uso com @ViewChild

Nota sobre [(p-value)]: O po-dynamic-form suporta two-way binding com [(p-value)] para objetos simples (não signals). Com signals, use [p-value]="formValues()" e inicialize o objeto antes de abrir o modal. Não existe (p-value-change) nesta versão do PO-UI — nunca use-o. Para reagir a mudanças de campo em tempo real, use (p-form) e assine form.valueChanges (ver Quirk #13 em po-ui-quirks.md).

PoDialogService

Serviço para diálogos de confirmação e alerta — sem componente no template.

confirm

alert

Nota: PoDialogService NÃO precisa de po-dialog no template. Ele cria o diálogo programaticamente via serviço. Injete apenas no inject(). Já está disponível como providedIn: 'root' — não requer declaração em providers.

po-dynamic-view

Exibe campos de um objeto em layout de grade — ideal para telas de detalhe/visualização somente leitura.

Key Inputs

InputTypeDescription
p-fieldsPoDynamicViewField[]Definição dos campos exibidos
p-valueobjectObjeto com os valores a exibir

PoDynamicViewField

Exemplo

Dica de uso combinado: Em modais de detalhe (sem edição), substitua o po-dynamic-form pelo po-dynamic-view com os mesmos fields — reusa a definição de campos sem duplicar código.

po-page-slide

Painel lateral deslizante a partir da borda direita da tela — alternativa ao po-modal quando o conteúdo secundário precisa coexistir visualmente com a página principal.

Key Inputs

InputTypeDescription
p-titlestringTítulo exibido no cabeçalho do painel
p-size'sm' | 'md' | 'lg' | 'xl'Largura do painel (default 'md')
p-hide-closebooleanOculta o botão de fechar
p-click-outbooleanFecha ao clicar fora do painel (default false)

Key Outputs

OutputPayloadDescription
(p-close)voidEmitido ao fechar o painel

Uso com @ViewChild

Exemplo completo

po-page-slide vs po-modal: Use po-page-slide quando o usuário precisa continuar vendo a página principal enquanto visualiza o detalhe. Use po-modal quando a tarefa exige atenção total.
