PO-UI Navigation Components

Botão de ação, menu lateral, toolbar, abas e componentes de navegação PO-UI.

PO-UI Navigation Components

po-button

Botão de ação padrão do PO-UI — componente mais usado em toda a biblioteca. Suporta quatro estilos visuais, loading integrado, ícone e type HTML.

Key Inputs

InputTypeDescription
p-labelstringTexto exibido no botão
p-iconstringÍcone PO (ex: 'po-icon-plus', 'po-icon-save', 'po-icon-delete')
p-kind'primary' | 'secondary' | 'tertiary' | 'link'Estilo visual (default 'secondary')
p-type'button' | 'submit' | 'reset'Tipo HTML do botão (default 'button')
p-disabledbooleanDesabilita o botão
p-loadingbooleanExibe spinner e desabilita temporariamente
p-size'medium' | 'large'Tamanho do botão (default 'medium')
p-dangerbooleanAplica cor de perigo (vermelho) — combina com p-kind="secondary"

Key Outputs

OutputPayloadDescription
(p-click)voidEmitido ao clicar no botão

PoButtonKind — estilos visuais

KindQuando usar
'primary'Ação principal da tela — apenas um por contexto
'secondary'Ações secundárias e de confirmação (default)
'tertiary'Ações de menor importância / destaque mínimo
'link'Ação com aparência de link de texto

Exemplos

Dica: Para ações em páginas (po-page-edit, po-page-detail), use p-actions: PoPageAction[] em vez de po-button manual — o PO-UI cuida do layout responsivo e do menu "Mais ações" automaticamente.

po-search

Campo de busca com autocomplete e filtragem de objetos por chave — uso standalone, fora de po-page-list.

Key Inputs

InputTypeDescription
p-labelstringRótulo ou placeholder do campo
p-placeholderstringTexto de ajuda (alternativa a p-label)
p-itemsany[] | string[]Itens para autocomplete
p-filter-keysstring[]Propriedades dos objetos usadas para filtrar
p-search-type'action' | 'input''action' = busca ao pressionar Enter · 'input' = busca a cada keystroke (default 'action')
p-disabledbooleanDesabilita o campo

Key Outputs

OutputPayloadDescription
(p-change-model)stringEmitido ao digitar ('input') ou ao submeter ('action')

Exemplos

po-search vs p-quick-search: Use p-quick-search no po-page-list para filtro integrado ao filtro avançado. Use po-search para busca standalone fora de uma po-page-list.

po-menu

Menu lateral navegável com suporte a grupos, ícones, badges, filtro e colapso. Normalmente declarado uma vez no shell/app component — não em cada tela.

Key Inputs

InputTypeDescription
p-menusPoMenuItem[]Array de itens do menu
p-collapsedbooleanInicia o menu recolhido (default false)
p-logostringURL do logotipo exibido no topo
p-logo-altstringTexto alternativo para o logo
p-filterbooleanExibe campo de filtro para buscar itens
p-short-logostringLogo menor exibido quando o menu está recolhido

Key Outputs

OutputPayloadDescription
(p-collapsed)booleanEmitido ao colapsar/expandir

PoMenuItem

Exemplo completo de menu lateral

Nota: po-menu usa [routerLink] internamente quando link é fornecido — requer provideRouter() no app.config.ts.

[p-menu-header-template]

Projeta conteúdo customizado na área acima dos itens do menu. Sem context variable — o template acessa o escopo do componente pai diretamente.

Usos comuns: user card (avatar + nome + papel), logo dinâmico por filial, seletor de empresa. O template re-renderiza junto com o CD do componente pai — reatividade funciona normalmente com Signals.

po-menu-panel

Painel de navegação lateral compacto — ícone-only quando recolhido, ícone + label quando expandido. Ideal quando o espaço horizontal é limitado ou para apps com poucos itens de navegação.

Key Inputs

InputTypeDescription
p-itemsPoMenuPanelItem[]Itens de navegação
p-logostringURL do logo exibido quando expandido
p-short-logostringURL do logo reduzido exibido quando recolhido

PoMenuPanelItem

Exemplos

po-menu vs po-menu-panel: po-menu é o menu lateral completo com grupos, sub-itens, filtro e badges — adequado para apps complexos. po-menu-panel é navegação compacta ícone-based, ideal para painéis e apps com poucos itens de menu.

po-toolbar

Barra de topo da aplicação com título, logo, perfil do usuário, notificações e ações customizadas.

p-itemsPoToolbarItem[]Ações adicionais (ícones com ação)
p-profilePoToolbarProfileDados do usuário logado
p-notificationPoToolbarNotificationConfiguração do ícone de notificações

po-stepper + po-step

Divide um processo em passos numerados — ideal para wizards de cadastro com múltiplas etapas que precisam ser completadas em sequência.

Key Inputs (po-stepper)

InputTypeDescription
p-stepnumberÍndice do passo ativo (1-based)
p-orientation'horizontal' | 'vertical'Layout do stepper (default 'horizontal')
p-sequentialbooleanImpede pular passos via clique no número (default false)

Key Outputs (po-stepper)

OutputPayloadDescription
(p-change-step)numberEmitido ao navegar entre passos

Key Inputs (po-step)

InputTypeDescription
p-labelstringRótulo do passo exibido no stepper
p-statusPoStepperStatusEstado visual do passo

PoStepperStatus

Exemplo

p-sequential: Use [p-sequential]="true" para bloquear o avanço por clique no número do passo — o usuário só navega pelos botões Próximo/Anterior.
