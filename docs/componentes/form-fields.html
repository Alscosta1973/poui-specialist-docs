[p-combo-option-template] — template de opção personalizado

Customiza a renderização de cada opção no dropdown. Use quando a opção precisa exibir mais que apenas o label — avatar, status, departamento, badges, etc.

Contexto do template: let-option expõe o objeto completo de p-options — inclua propriedades extras como avatarUrl, departamento, status além de label e value.

po-datepicker

po-datepicker-range

Campo para seleção de um período entre duas datas — início e fim em um único campo. Ideal para filtros de relatório, parametrização de períodos ou intervalo de datas em formulários.

Key Inputs

InputTypeDescription
p-labelstringRótulo do campo
p-min-dateDate | stringData mínima permitida
p-max-dateDate | stringData máxima permitida
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o campo
p-cleanbooleanExibe botão para limpar o valor (default true)

Key Outputs

OutputPayloadDescription
(p-change)PoDatepickerRangeEmitido ao selecionar o intervalo

PoDatepickerRange

Exemplos

Reactive Forms: O valor do formControl é PoDatepickerRange — ao submeter, acesse form.value.periodo.start e form.value.periodo.end.

po-datetimepicker

Campo que combina data e hora em um único input — substitui o uso de po-datepicker + po-time juntos. Útil para agendamentos e registros de eventos com hora exata.

Key Inputs

InputTypeDescription
p-labelstringRótulo do campo
p-min-datestringData/hora mínima (formato ISO yyyy-MM-ddTHH:mm:ss)
p-max-datestringData/hora máxima
p-no-secondsbooleanOculta os segundos no campo de hora (default false)
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o campo

Key Outputs

OutputPayloadDescription
(p-change)stringData/hora selecionada em formato ISO

Exemplos

Substitui po-datepicker + po-time: Use po-datetimepicker sempre que precisar de data e hora no mesmo campo — evita sincronizar dois campos separados no formulário.

po-textarea

po-upload

Componente para envio de arquivos. Suporta upload automático ou manual, múltiplos arquivos, drag-and-drop, e validação de extensão e tamanho.

Key Inputs

InputTypeDescription
p-urlstringURL de destino do upload (multipart/form-data)
p-allowed-extensionsstring[]Extensões permitidas (ex: ['.pdf', '.jpg', '.png'])
p-auto-uploadbooleanInicia upload imediatamente ao selecionar o arquivo
p-drag-dropbooleanHabilita zona de drag-and-drop
p-multiplebooleanPermite selecionar múltiplos arquivos
p-max-file-sizenumberTamanho máximo em bytes (ex: 5242880 = 5 MB)
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o componente
p-labelstringRótulo exibido acima do componente

Key Outputs

OutputPayloadDescription
(p-upload)PoUploadFile[]Emitido após upload bem-sucedido
(p-success)anyResposta HTTP do servidor após upload
(p-error)anyErro de upload (HTTP ou validação)
(p-change)PoUploadFile[]Arquivos selecionados (antes do upload)

Exemplos

Integração Protheus: O endpoint de upload deve aceitar multipart/form-data. No ADVPL, use oRequest:GetMultipartBody() para ler o arquivo recebido.

po-time

Campo de entrada de hora com máscara automática — formato HH:MM ou HH:MM:SS.

Key Inputs

InputTypeDescription
p-labelstringRótulo do campo
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o campo
p-no-secondsbooleanOculta os segundos — usa formato HH:MM (default false)
p-min-timestringHora mínima permitida (formato HH:MM ou HH:MM:SS)
p-max-timestringHora máxima permitida

Exemplos

po-timepicker

Componente de hora com painel flutuante de seleção — API mais recente que po-time. Prefira po-timepicker em novos componentes; po-time segue suportado mas não evolui.

Key Inputs

InputTypeDescription
p-labelstringRótulo do campo
p-no-secondsbooleanOculta os segundos — usa formato HH:MM (default false)
p-min-timestringHora mínima permitida (formato HH:MM ou HH:MM:SS)
p-max-timestringHora máxima permitida
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o campo

Key Outputs

OutputPayloadDescription
(p-change)stringHora selecionada em formato HH:MM:SS ou HH:MM

Exemplos

po-time vs po-timepicker: Ambos funcionam, mas po-timepicker é a API mais recente com painel flutuante de seleção. Use po-timepicker em novos componentes.

po-chips

Campo de entrada de múltiplos valores como chips — ideal para tags, e-mails, palavras-chave e filtros. Cada valor digitado vira um chip removível; o formulário recebe string[].

Key Inputs

InputTypeDescription
p-labelstringRótulo do campo
p-placeholderstringTexto de ajuda antes de digitar
p-separatorstringCaractere que confirma um chip além do Enter (ex: ',')
p-requiredbooleanCampo obrigatório
p-disabledbooleanDesabilita o campo

Key Outputs

OutputPayloadDescription
(p-change)string[]Emitido ao adicionar ou remover um chip

Exemplos

Valor no formulário: formControl.value é string[]. Para pré-popular, inicialize o controle com um array: new FormControl(['Angular', 'TypeScript']).
