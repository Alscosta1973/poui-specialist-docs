PO-UI Layout Components

po-container

Agrupa conteúdo com borda, título e espaçamento interno — ideal para separar seções de formulário sem usar divider no po-dynamic-form, especialmente quando cada seção tem sua própria lógica.

Key Inputs

InputTypeDescription
p-titlestringTítulo exibido no topo do container
p-no-borderbooleanRemove a borda (mantém padding)
p-no-paddingbooleanRemove o padding interno
p-heightnumberAltura fixa em pixels (scroll interno se o conteúdo exceder)

Exemplos

po-accordion + po-accordion-item

Agrupa conteúdo em seções retráteis — útil para formulários longos com muitas seções, ou telas de detalhe onde o usuário precisa explorar seções sob demanda.

po-accordion — Key Inputs

InputTypeDescription
p-show-manager-accordionbooleanExibe botões "Expandir todos / Recolher todos"
p-allow-expand-all-itemsbooleanPermite múltiplas seções abertas simultaneamente (default: somente uma)
p-size'medium' | 'large'Tamanho visual (default 'medium')

po-accordion-item — Key Inputs

InputTypeDescription
p-labelstringRótulo do item / título da seção
p-disabledbooleanDesabilita o item (impede expansão)

po-list-view

Lista card-style com templates customizáveis — alternativa ao po-table para dados mais visuais, como listas de documentos, produtos com imagem, ou registros com muita informação contextual.

Key Inputs

InputTypeDescription
p-itemsany[]Array de dados
p-property-titlestringPropriedade usada como título de cada card
p-property-linkstringPropriedade usada como link do título
p-loadingbooleanIndicador de carregamento
p-show-more-disabledbooleanDesabilita o botão "Mostrar mais"
p-selectablebooleanHabilita seleção de itens com checkbox
p-hide-select-allbooleanOculta o checkbox "Selecionar todos"

Key Outputs

OutputPayloadDescription
(p-show-more)voidClique em "Mostrar mais" — carregar próxima página
(p-selected)any[]Array de itens selecionados

po-list-view vs po-table — quando usar qual

Critériopo-tablepo-list-view
Dados tabulares (muitas colunas)✅ Ideal⚠️ Limitado
Cards visuais / layout livre⚠️ Limitado✅ Ideal
Ordenação por coluna✅ Nativo❌ Manual
Templates de célula customizados✅ via diretiva✅ via template
Detalhe expandível (master-detail)✅ type: 'detail'✅ detail-template
Mobile-friendly por padrão⚠️ Scroll horizontal✅ Cards empilham
